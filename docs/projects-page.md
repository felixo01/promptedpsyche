# Projects Page

## Purpose

The Projects page exists at:

- `/projects/`
- `/pl/projects/`

It gives a public overview of Prompted Psyche project directions: research-informed work, applied inquiry, prototypes, conceptual models and tools for thinking.

The page should make Feliks Mamczur's work legible without turning unfinished directions into product promises.

## Public Naming Rules

Use public descriptive names unless a project name is intentionally launched.

Internal working names should stay backstage by default. They should not appear as public project titles unless there is a separate launch decision.

Current public names:

- `AI and psychoeducation` / `AI i psychoedukacja`
- `AI-assisted diagnostic reasoning` / `AI jako wsparcie rozumowania diagnostycznego`
- `HumanAI research`
- `Prompted Psyche Concepts` / `Słownik Prompted Psyche`

## Card Structure

Public project cards use:

- title
- status
- description
- what it concerns
- what it does not do
- optional link to an existing public section

## Safety Rules

The Projects page must not include:

- product claims
- diagnosis or therapy claims
- claims that AI replaces specialist responsibility
- architecture details
- prompts
- roadmap
- MVP details
- private repository links
- partner names or client names

Project descriptions can support future academic credibility, but they must not claim institutional status, peer review or formal research output unless that becomes true and sourceable.
